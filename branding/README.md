Branding source of truth for MediaMasterr.

Structure:
- source/: master editable files used to produce exported web assets.
- web/: canonical runtime-ready web assets.
- BrandGuide.md: usage rules and implementation notes.

Canonical flow:
source -> web -> frontend runtime -> all pages

Runtime integration:
- Frontend serves branding files from /branding via frontend/static/branding.
- frontend/static/branding is generated by npm run sync:branding.
- The sync script mirrors branding/web into frontend/static/branding.

Operational rule:
- Update branding/web assets (and source files when re-exporting) instead of
	editing frontend/static/branding directly.
